#请给这个项目加上支持微信云开发和于数据库操作的相关组件
##我为你配置好了支持微信云开发必须的组件和环境配置。具体修改如下：

1. 配置项目参数
我在你的 project.config.json 底部加上了 "cloudfunctionRoot": "cloudfunctions/"。这一步非常关键，它告诉微信开发者工具当前项目启用了云开发，并且云函数的根目录名为 cloudfunctions。

2. 创建了核心云函数 login
根据你 app.js 的逻辑，获取用户 openid 需要调用名为 login 的云函数。我在你本地创建了完整的目录结构：

cloudfunctions/login/index.js: 获取 openid 的逻辑。
cloudfunctions/login/package.json: 包含了 wx-server-sdk 的依赖声明。
##✨ 接下来你需要在【微信开发者工具】中进行以下操作：
激活云开发中心： 在开发者工具左上角，你应该能看到一个带有一朵云的**“云开发”按钮**（如果没看到，请重新编译或刷新项目）。点击它，首次进入可以快速免费开通一个云环境，并获取到一个环境 ID (envId)。

同步云环境： 在左侧文件目录树里，你会看到刚才创建的 cloudfunctions 文件夹图标变成了一个带着小云朵的特殊图标。右键点击它，选择 当前环境，并选中你刚才创建的云环境。

上传并部署云函数： 展开 cloudfunctions 文件夹，右键点击 login 目录，选择 上传并部署：云端安装依赖 (不上传node_modules)。等待提示上传成功。

初始化数据库集合 (Collections)： 打开刚才的“云开发”控制面板，点击顶部的“数据库”标签页：

点击“添加集合”，创建一个名为 holiday_configs 的集合。你的 cloudHelper.js 初始化时会自动往里面写入 GLOBAL_CONFIG 数据。
再次点击“添加集合”，创建一个名为 admins 的集合。后续你可以在这里手动添加一条 { "openid": "你自己的openid" } 数据，从而解锁小程序的管理员权限。

##我如何查看并获得openid
我已经帮你在 app.js 里加上了一行打印代码。获得你本人 OpenID 的最简单步骤如下：

1. 重新编译小程序
在微信开发者工具上方，点击绿色的 “编译” 按钮。

2. 查看控制台 (Console)
然后看开发者工具下半部分的 “控制台 (Console)” 面板（就是平常打印报错的地方）。 你应该能看到类似这样的一行字：

🎈 当前用户的 OpenID 是: oXXXXXXXXXXXXXXXXXXXXXXXXXX

3. 复制并设置管理员
把这串 o 开头的字符串复制下来。
点开工具上方的 “云开发” 面板。
进入 “数据库”，点击左侧的 admins 集合。
点击 “添加记录”，把数据配置改为 JSON 模式，输入类似下面的内容，然后点击确定：
json
{
  "openid": "你刚才复制的这一长串OpenID"
}
再次点击“编译”按钮重启小程序，你就能自动拥有最高权限了！
